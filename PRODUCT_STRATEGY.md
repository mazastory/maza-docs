## 1. 핵심 포지셔닝

### 기존 (폐기)
> ~~"애드센스로 돈 벌어드립니다"~~  
> 수익을 약속하는 카피는 클레임 리스크 + 신뢰도 하락으로 사용 금지.

### 확정 포지션
> **"콘텐츠 운영을 자동화해주는 1인 미디어 인프라"**

### 핵심 카피 (Hero)
> **"주 5개 글을 30분 작업으로 — 발행 + 다국어 + 배포까지 끝낸다"**

### 차별화 메시지 3가지
1. **"써놓으면 끝"** — 글 작성 후 배포/사이트맵/광고삽입/sitemap 전부 자동
2. **"정책 위반 걱정 없음"** — 발행 속도·법적페이지·AI 고지가 시스템에 내장
3. **"한 번 쓰면 3개 언어"** — 다국어 자동확장 (Pro부터)

## 2. 핵심 타겟 고객

### 1차 타겟 (집중)
> **현재 티스토리·네이버 블로그를 운영 중이지만 한계를 느끼는 한국 직장인 개인**

- 이미 글 쓰는 습관이 있음
- 애드센스·수익화 개념을 이미 알고 있음
- 현재 플랫폼의 제약(수익 한계, 플랫폼 종속)에 불만 있음
- "도구를 배우는 데" 시간 쓰기 싫음 → 자동화에 즉각 반응하는 군

### 2차 타겟 (6개월+ 이후)
- 소규모 콘텐츠 에이전시 / 프리랜서 에디터
- 해외 수익화를 원하는 크리에이터

### 하지 말아야 할 것
- 처음부터 "글로벌+국내 다 잡기" 시도
- 기업 B2B 타겟팅 (초기 운영 부담 과중)

## 3. 플랜 & 가격 (확정)

| 플랜 | 가격 | 서브도메인 | 언어 | 월 발행 한도 | 특징 |
|------|------|-----------|------|------------|------|
| **Starter** | **월 9,900원** | 1개 | 한국어만 | 8개 (주 2개) | 법적 페이지 자동생성, 애드센스 자동삽입 |
| **Pro** | **월 49,000원** | 3개 | 한국어 + 영어·일본어 자동번역 | 무제한 | 쿠팡파트너스 자동삽입, 다국어 SEO 최적화 |
| **Agency** | **월 99,000원** | 무제한 | 전체 | 무제한 | 통합 대시보드, 화이트라벨(고객사 재판매 가능) |

### 가격 전략 노트
- **9,900원** — 심리적 "1만원 미만" 진입점. 커피 2잔 가격으로 티스토리 탈출 가능하다는 메시지.
- **49,000원** — Pro는 월 5만원 미만. 다국어가 핵심 차별점으로 소구.
- **99,000원** — Agency는 10만원 미만. 에이전시가 고객사에 재판매 시 마진 충분.
- **Revenue Share 모델 제외** — 애드센스 수익은 유저 계정으로 직접 입금되므로 정산 인프라 구축 비용이 초기 매출보다 먼저 발생. 구독 단일 모델로 시작 → 안정화 이후 Pro 이상 고객 대상 옵션 추가 검토.

### 도입 프로모션 (옵션)
- 첫 1개월 무료 체험 (카드 등록 필요) — 온보딩 완료율 높이기 위한 장치
- Agency는 초기 "출시 예정" 배지 + 이메일 웨이팅 리스트로 시작 (너무 일찍 판매 시 요구사항으로 로드맵 왜곡 위험)

## 4. 제품 기능 — 발행 속도 가드레일

### 핵심 원칙
> Starter의 **월 8개(주 2개) 발행 한도**를 DB 레벨에서 강제.  
> 이를 **"제약"이 아닌 "구글 정책 자동 준수 기능"으로 마케팅**.

### 마케팅 프레이밍
- ❌ "월 8개까지만 발행 가능합니다"  
- ✅ "구글 스팸 정책을 자동으로 준수해 드립니다. 유저가 직접 신경 쓸 필요 없습니다"

### 플랜별 발행 한도
| 플랜 | 월 한도 | 일일 한도 | 적용 방식 |
|------|--------|---------|---------|
| Starter | 8개 | 1개 | DB 레벨 강제 |
| Pro | 무제한 | 3개 권장 | 소프트 경고 |
| Agency | 무제한 | 제한 없음 | 모니터링만 |

## 5. MVP 4주 개발 로드맵

### 1주차 — 애드센스 신청 가능한 상태 만들기
- [ ] DB 테이블명 정리 (`ms_sites`, `ms_posts` 통일)
- [ ] 법적 페이지 4종 자동생성 (개인정보처리방침, 이용약관, 저작권 고지, 광고 고지)
- [ ] RSS 피드 경로 표준화 (`/rss.xml`)
- [ ] Sitemap 자동생성 및 경로 확정 (`/sitemap.xml`)

### 2주차 — 단일 사이트 완성도 끌어올리기
- [ ] API 라우트 정리 (불필요한 엔드포인트 제거)
- [ ] 광고 슬롯 위치 표준화 (본문 상·중·하)
- [ ] AI 생성 콘텐츠 고지 문구 자동삽입
- [ ] 쿠팡파트너스 링크 자동치환 로직 (Pro 전용)

### 3~4주차 — 다국어 (Pro 핵심 차별 기능)
- [ ] `/[lang]/[slug]` 라우팅 구조 확립
- [ ] 영어(en) 자동번역 API 연동 (1개 언어 먼저)
- [ ] 다국어 Sitemap 자동생성
- [ ] hreflang 태그 자동주입

> ⚠️ **주의**: 다국어 라우팅을 1주차에 건드리면 현재 코드 구조 전체가 흔들림. 반드시 2주차 이후로 미룰 것.

## 6. 핵심 KPI (초기 6개월)

| 지표 | 목표 | 이유 |
|------|------|------|
| 온보딩 완료율 | **≥ 80%** | 가입 후 첫 글 배포까지 완료한 비율. 가장 중요한 초기 지표 |
| Starter → Pro 전환율 | **≥ 15%** | 다국어 기능이 킬러 업셀 포인트 |
| 월 이탈률(Churn) | **≤ 5%** | 자동화 가치 체감 = 낮은 이탈 |
| NPS | **≥ 40** | "주변에 추천하겠는가" — 입소문 성장의 선행지표 |

## 8. 기술 스택 (확정)

| 레이어 | 기술 | 비고 |
|--------|------|------|
| 블로그 엔진 | Astro (SSG) | 구글 Core Web Vitals 최적화 |
| 호스팅 | Netlify | ZeroIT 자동 배포 파이프라인 |
| DNS | Cloudflare | CDN + 보안 |
| 코드 저장소 | GitHub | 유저별 자동 생성 |
| 백엔드 | Node.js + Express | Netlify 배포 |
| DB | Supabase (PostgreSQL) | 인증 + 데이터 |
| AI | Gemini / GPT | 콘텐츠 자동생성 |
| 마케팅 랜딩 | maza-site (Astro) | Netlify 배포 |
| 관리 앱 | maza-studio (React) | Netlify 배포 |
| 확장 프로그램 | Maza Extension (Chrome) | 티스토리 등 외부 블로그 DOM 자동화 인젝션 |

## 9. 프로젝트 구조 & 역할

```
MazaWorkspace/
├── maza-site/      # 마케팅 랜딩 페이지 (잠재 고객 유입)
├── maza-studio/    # 유저 관리 앱 + 백엔드 API + 크롬 익스텐션 (메인 SaaS)
├── maza-blog/      # 유저 블로그 엔진 템플릿 (ZeroIT로 자동 배포)
└── maza-docs/      # 내부 문서 & 유저 가이드
```

| 프로젝트 | 비유 | 대상 |
|---------|------|------|
| maza-site | 카페24 회사 홈페이지 | 서비스를 모르는 잠재 고객 |
| maza-studio | 카페24 관리자 콘솔 및 플러그인 | Maza 가입 유저 |
| maza-blog | 카페24가 만들어주는 쇼핑몰 엔진 | 유저들의 블로그 방문자 |
