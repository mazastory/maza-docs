# 🛡️ Maza Studio 강력한 CS 및 장애 대응 프로세스 (Playbook)

어드민(관리자) 입장에서 사이트 운영 중 발생할 수 있는 주요 문제들을 **가장 빠르고 정확하게 진단하고 대처할 수 있는 표준 프로세스**입니다. Maza Studio의 기능을 활용하여 강력한 고객 지원(CS)을 수행할 수 있습니다.

---

## 1. 애드센스 승인 거절 (가치 없는 콘텐츠 / 정책 위반)

가장 빈번하게 발생하는 고객 문의입니다. 구글 봇이 사이트를 스캔했을 때 콘텐츠의 질이 떨어지거나 기술적 결함이 있을 때 발생합니다.

### 🔍 진단 프로세스 (How to identify)
1. **[CS 관제탑]** 에서 고객 이메일 검색 후 **[QC 점검]** 버튼 클릭.
2. **SEO 클리닉 정밀 스캔** 결과를 확인합니다.
   * `글자수 1300자 미만` 또는 `이미지 3개 미만` 게시글이 있는지 확인 (Thin Content).
   * `H2/H3 태그 구조`가 무너져 있는지 확인.
   * 사이트 내 빈 카테고리나 깨진 링크(404)가 있는지 확인 (구글 봇이 혐오함).

### 🛠️ 대처 프로세스 (How to fix)
* **경증 (일부 글만 문제일 때):** 관제탑에서 **[자동 복구] (Auto Fix)** 버튼을 클릭합니다. AI가 짧은 글에 '관련 배경 지식(위키백과 등)'을 추가로 주입하여 글자수(EEAT)를 보강하고 누락된 이미지를 자동으로 백필(Backfill)합니다.
* **중증 (글 품질이 전반적으로 심각할 때):** 
  * 기준 미달(V2 Score 70점 미만) 게시글은 수동 복구하지 말고 과감히 **삭제**합니다.
  * Maza Studio의 **Self-Healing(자가 치유)** 파이프라인이 삭제된 키워드 대신 연관된 새로운 고품질 키워드를 발굴하여 큐(Queue)에 다시 올려 자동 발행하도록 둡니다.
* **고객 응대 스크립트:** *"고객님, 현재 사이트 내 일부 게시글의 분량이 구글 기준에 미달하여 AI 자동 복구 모드를 가동했습니다. 1~2일 뒤 콘텐츠가 빵빵하게 채워지면 애드센스 재심사를 눌러주세요!"*

## 2. 구글 인프라 연동 누락 (애드센스 / 서치콘솔)

고객이 초기 설정을 건너뛰었거나, 테마를 변경하다가 필수 태그(`pagead2`, `google-site-verification`)가 날아간 경우입니다. 트래픽이 안 잡히고 애드센스 심사 자체가 시작되지 않습니다.

### 🔍 진단 프로세스
1. **[CS 관제탑]** 에서 사이트 검색 시 **구글 인프라 검증** 박스 확인.
2. 서치콘솔 코드 또는 애드센스 `pub-ID`가 '없음'으로 뜨는지 확인.
3. `ads.txt` 누락 또는 `pub-ID 불일치` 경고(노란색 또는 빨간색)가 떠 있는지 확인.

### 🛠️ 대처 프로세스
* **Maza 플랫폼(자체 호스팅):** 어드민이 고객의 `pub-ID`만 받아서 DB(`sites` 테이블)에 직접 입력해주면, SSR 렌더링 엔진(`Layout.astro`)이 즉시 사이트 전체 `<head>`와 `ads.txt`에 태그를 전파합니다.
* **워드프레스/티스토리:** 고객에게 "Maza Studio 대시보드 - 구글 연동 메뉴"에서 서치콘솔과 애드센스 ID를 다시 한 번 저장(Save) 버튼을 누르도록 안내합니다 (API를 통해 원격으로 헤더에 삽입됨).

## 3. AI 자동 발행 멈춤 (무한 대기 현상)

"글이 안 올라와요", "스케줄이 멈췄어요" 라는 문의가 올 때의 대응입니다.

### 🔍 진단 프로세스
1. **[CS 관제탑]** 의 **AI 발행 대기열 (Queue)** 박스를 확인.
2. `FAILED` 숫자가 높거나, `GENERATING` 상태에서 숫자가 오랫동안 멈춰있는지(Zombie Lock) 확인.
3. **API 할당량 및 격리 상태** 박스에서 해당 사이트가 `QUARANTINED` (격리/차단) 상태인지 확인 (예: 구글 API 429 Quota 초과 또는 403 밴).

### 🛠️ 대처 프로세스
* **Zombie Lock 해제:** 어드민 대시보드의 백그라운드 크론(Cron) 스크립트가 멈춘 작업을 감지하면 `started_at`을 롤백하고 상태를 `QUEUED`로 되돌리지만, 강제 처리가 필요할 경우 DB에서 해당 게시글의 `worker_locked`를 `false`로 풀어줍니다.
* **API Key 교체 (Rotation):** 구글 제미나이(Gemini) 할당량을 다 썼다면, 다른 계정의 API 키를 적용하거나 자정이 지나 쿼터가 리셋될 때까지 기다려야 합니다. (이때 무리하게 재시도 버튼을 연타하면 안 됩니다).
* **고객 응대 스크립트:** *"현재 구글 AI 트래픽이 폭주하여 대기열에 병목이 생겼습니다. 글이 증발한 것이 아니라 대기 중이니, 오늘 밤~내일 아침 사이 순차적으로 발행 완료될 예정입니다."*

## 4. 사이트 분양 및 소유권 이관

사이트 제작이 완료되어 최종 고객(광고주/블로거)에게 사이트를 넘길 때의 프로세스입니다.

### 🔍 진단 프로세스
* 고객이 Maza Studio 회원가입(구글 로그인)을 완료했는지 이메일을 통해 확인합니다.

### 🛠️ 대처 프로세스
* **[CS 관제탑]** -> 타겟 사이트 옆의 🌐 **[이관하기]** 클릭.
* 고객의 이메일 주소 입력 후 승인.
* **(자동 처리 내용):** 기존에 박혀있던 대표님(어드민)의 애드센스 코드, 서치콘솔 태그가 즉시 깔끔하게 폭파(초기화)되고 소유권이 고객에게 100% 넘어갑니다. 구글 정책 위반(계정 중복) 리스크를 원천 차단합니다.
