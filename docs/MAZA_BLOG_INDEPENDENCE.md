# 마자 블로그 독립(Independence) 및 데이터 이관 가이드

## 개요
SaaS(Software as a Service) 환경에서 사용자가 가장 우려하는 것은 **데이터 종속성(Lock-in)**입니다. "구독을 해지하면 내 블로그와 정성껏 작성한 글들이 모두 날아가는 것 아닐까?"라는 불안감을 해소하기 위해, 마자 스튜디오는 완벽한 **데이터 소유권 이전 및 블로그 독립 시스템**을 제공합니다.

구독이 종료되더라도 사용자는 본인의 데이터베이스(Supabase)를 통해 평생 무료로 마자 블로그를 소유하고, 내장된 수동 에디터를 통해 글을 계속 작성할 수 있습니다.

---

## 🙋‍♂️ 자주 묻는 질문 (FAQ)

### Q. 구독을 해지하면 제 블로그와 글들은 모두 삭제되나요?
**A. 전혀 그렇지 않습니다!** 
언제든 원클릭으로 고객님의 데이터를 빼내어 개인 데이터베이스로 완벽히 독립할 수 있습니다. 마자 블로그에는 평생 무료로 운영하실 수 있는 **'자체 수동 글쓰기 에디터'**가 내장되어 있으므로, 구독이 종료되어도 예쁜 마자 블로그 템플릿을 그대로 사용하며 영구적으로 블로그를 소유하실 수 있습니다.

### Q. 이관 후에는 어떻게 글을 쓰나요?
**A. 마자 블로그에 내장된 전용 에디터를 사용합니다.**
독립된 블로그 도메인의 `/admin` 경로로 접속하시면, 본인의 DB와 연결된 관리자 대시보드가 나타납니다. 여기서 기존 글을 수정/삭제하거나, 실시간 미리보기(Live Preview)가 지원되는 마크다운(Markdown) 에디터로 새로운 글을 발행할 수 있습니다.

## 🚀 블로그 독립 3단계 가이드

마자 스튜디오 [내 사이트(Site Detail)] 화면 우측 상단의 **[내 블로그 독립 (데이터 내보내기)]** 버튼을 누르시면 아래의 절차를 안내받으실 수 있습니다.

### 1단계: 무료 데이터베이스(Supabase) 생성
1. [Supabase](https://supabase.com)에 가입하고 무료 프로젝트를 하나 생성합니다.
2. 이 공간이 앞으로 고객님의 블로그 데이터가 영구적으로 보관될 장소입니다.

### 2단계: 테이블 생성 및 데이터 백업
1. 마자 스튜디오 팝업에서 **[DB 스키마 다운로드 (SQL)]** 버튼을 눌러 스크립트 파일을 받습니다.
2. 생성하신 Supabase의 'SQL Editor' 메뉴에 코드를 붙여넣고 실행하여 `sites`, `posts` 테이블을 생성합니다.
3. 마자 스튜디오 팝업에서 **[전체 게시글 백업 (JSON)]** 버튼을 눌러 지금까지 작성한 모든 글을 다운로드한 후, 본인의 Supabase에 Import 합니다.

### 3단계: 블로그(Netlify) 연결 정보 변경
1. [Netlify](https://app.netlify.com) 대시보드에 로그인하여 내 블로그 프로젝트에 접속합니다.
2. `Site Configuration > Environment variables` 메뉴로 이동합니다.
3. 다음 두 가지 환경변수를 본인이 생성한 Supabase의 정보로 교체합니다:
   - `PUBLIC_SUPABASE_URL` = 나의 Supabase Project URL
   - `PUBLIC_SUPABASE_ANON_KEY` = 나의 Supabase API Key
