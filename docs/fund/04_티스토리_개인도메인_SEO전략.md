# 티스토리 개인 도메인 연결 시 SEO 및 서치콘솔 가이드

티스토리(예: `zerowork.tistory.com`) 블로그를 운영하다가 개인 도메인(예: `autosite.kr`)을 연결하게 되면, 검색 엔진 최적화(SEO)와 사이트 운영 측면에서 기존 주소에서 새 주소로의 이전을 알리기 위해 여러 가지 설정 변경이 필요합니다.

많은 유저분들이 궁금해하시는 **"도메인 변경 시 필수 체크리스트와 사이트맵/RSS 처리 방법"**을 정리했습니다.

---

## 1. 🔍 검색 엔진은 새 도메인을 "완전히 새로운 사이트"로 인식합니다.

가장 중요하게 명심해야 할 점입니다. 기존에 티스토리 주소로 글을 20개를 썼든 100개를 썼든, 개인 도메인을 입히는 순간 구글(서치콘솔)과 네이버(서치어드바이저) 입장에서는 **"완전히 새로운 깡통 사이트가 생겼다"**고 판단합니다. 

따라서 **모든 검색 엔진 도구에 '새로운 사이트'를 등록하고 세팅하는 과정을 처음부터 다시 진행**해야 합니다.

---

## 2. 📝 기존 글들의 사이트맵 및 RSS 반영

**Q. 기존에 20개의 글을 작성한 상태에서 도메인을 연결하면, 사이트맵과 RSS는 21번째 글부터 인식되나요?**

**A. 아닙니다. 기존 글(1~20번째 글)들도 모두 새 도메인으로 인식됩니다.**

개인 도메인을 연결하는 순간, 티스토리 시스템이 자동으로 기존에 작성했던 글들을 모두 포함하여 새로운 도메인 기준의 사이트맵과 RSS를 생성합니다. 

`https://autosite.kr/sitemap.xml`에 접속해 보면 아래와 같이 예전 글들도 모두 새 도메인 주소로 바뀌어 목록에 포함된 것을 확인할 수 있습니다.
* `https://autosite.kr/1` (1번째 글)
* `https://autosite.kr/2` (2번째 글)
* ...
* `https://autosite.kr/20` (20번째 글)

따라서 도메인 연결 전에 쓴 글이라고 해서 검색에서 누락되지 않으며, 과거의 글들도 모두 새 도메인 주소로 검색 엔진에 노출됩니다.

---

## 3. ⚙️ 필수 체크리스트 및 설정 변경 가이드

개인 도메인을 연결한 후에는 아래 항목들을 순서대로 반드시 재설정해야 합니다.

### ① 웹마스터 도구 (Search Console / 서치어드바이저) 재등록
기존 티스토리 주소로 1개의 글을 썼을 때 사이트맵과 RSS를 제출했더라도, 새 도메인 연결 후에는 **무조건 새로 등록**해야 합니다.

1. 서치콘솔 및 서치어드바이저에 접속하여 **새로운 속성 추가**로 `autosite.kr`을 등록합니다.
2. 소유권 확인(HTML 메타 태그 추가 등)을 다시 진행합니다.
3. 새로 만든 `autosite.kr` 속성 메뉴에서 **사이트맵(`sitemap.xml`)과 `rss`를 새로 제출**합니다.

> 💡 **꿀팁 (주소 이전 기능 사용하기):**
> 새로운 속성(`autosite.kr`)을 등록하신 후, 기존 속성(`zerowork.tistory.com`)의 서치콘솔 설정 메뉴에 가보시면 **[주소 변경]** 이라는 기능이 있습니다. 여기서 "내가 새 도메인으로 이사했다"라고 구글에 신고해 주시면, 기존 티스토리 도메인으로 쌓아두었던 검색 노출 점수(SEO)를 새 도메인으로 더 빠르고 안전하게 넘겨줄 수 있습니다.

### ② 구글 애드센스 (Google AdSense)
애드센스로 수익을 창출하고 계셨다면 다시 승인 과정을 거쳐야 합니다.
* **사이트 추가 및 승인:** 새 도메인(`autosite.kr`)을 애드센스의 '사이트' 메뉴에 새로 추가하고 **새로 승인(애드고시)**을 받아야 합니다. 티스토리 도메인으로 승인받은 하위 도메인이 아니기 때문에 별도의 심사가 필요합니다.
* **ads.txt 문제 해결:** 최근 티스토리에서는 [수익] 메뉴에서 ads.txt 연동을 지원하므로 해당 설정이 정상적으로 연결되어 있는지 확인합니다.

### ③ 방문자 분석 도구 (Google Analytics 등)
* 구글 애널리틱스(GA4) 관리자 설정에서 속성의 기본 URL을 기존 티스토리 주소에서 새 개인 도메인 주소로 변경해야 합니다. 이를 통해 유입 데이터가 정상적으로 새 도메인 기준으로 측정됩니다.

### ④ 메타데이터 및 HTML 헤드 (Canonical 태그)
티스토리 관리자 설정에서 개인 도메인을 정상적으로 연결(보안 접속 인증서 발급 완료)하면, 티스토리가 자동으로 HTML의 `<link rel="canonical" href="...">` 태그를 새 도메인 기준으로 변경해 주어 검색 엔진의 '중복 문서' 페널티를 방지해 줍니다. 
* 적용 후 브라우저에서 `F12`(개발자 도구)를 누르고 `<head>` 태그 안에 `canonical` 주소가 새 도메인으로 잘 설정되어 있는지 확인만 하시면 됩니다.

### ⑤ 스킨 내 하드코딩된 링크 수정
* 티스토리 스킨(HTML/CSS) 편집을 통해 메뉴나 로고 등에 기존 티스토리 링크를 직접 적어두셨다면, 모두 새 도메인으로 찾아 바꾸셔야 합니다.
* 본문 글을 작성할 때 삽입했던 이전 주소의 내부 링크들도 여유가 되실 때 새 주소로 수정하시는 것이 좋습니다.
