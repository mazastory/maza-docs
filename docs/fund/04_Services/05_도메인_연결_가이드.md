# 🌐 개인 도메인 연결 가이드 (DNS 설정)

Maza Studio에서 사이트 배포를 성공적으로 마치셨다면, 마지막으로 구매하신 도메인을 실제 사이트와 연결하는 작업이 필요합니다. 
이 과정은 도메인을 구입하신 사이트(가비아, 호스팅케이알 등)에 접속하셔서 **DNS 레코드**를 딱 2줄만 추가해 주시면 됩니다.

---

## 🎯 공통 필수 입력 정보

도메인 구입처에 상관없이, 우리가 입력해야 할 핵심 정보는 아래 두 가지입니다.

### 1. 루트 도메인 (예: `mysite.com`)을 쓸 경우
*   **A 레코드 추가**
    *   **타입/종류:** `A`
    *   **이름/호스트:** `@` (또는 빈칸, 또는 자신의 도메인명)
    *   **값/IP주소:** `75.2.60.5` (넷리파이 고정 IP)
*   **CNAME 레코드 추가 (www 접속용)**
    *   **타입/종류:** `CNAME`
    *   **이름/호스트:** `www`
    *   **값/목적지:** `[부여받은 넷리파이 주소].netlify.app`

### 2. 서브 도메인 (예: `blog.mysite.com`)을 쓸 경우
서브 도메인은 IP 주소(A 레코드)가 필요 없고 CNAME 딱 1줄만 추가하면 됩니다.
*   **CNAME 레코드 추가**
    *   **타입/종류:** `CNAME`
    *   **이름/호스트:** `blog` (원하는 서브도메인 이름)
    *   **값/목적지:** `[부여받은 넷리파이 주소].netlify.app`

---

## 🚀 [심화] 티스토리 사용자의 마자 독립 블로그 확장 및 이사 전략

이미 애드센스 승인을 받은 개인 도메인 티스토리 블로그(`autosite.kr`)를 운영 중이신가요? 티스토리의 한계를 벗어나 마자 독립 블로그로 확장하거나 완전히 이사(독립)할 때의 완벽한 DNS 설정 전략을 알려드립니다.

### 시나리오 A: 티스토리와 독립 블로그 '동시 운영' (서브도메인 확장)
티스토리 본진(`autosite.kr`)은 그대로 살려두고, 서브도메인(`maza.autosite.kr`)만 마자 독립 블로그로 세팅하여 수익 파이프라인을 2배로 늘리는 가장 추천하는 방법입니다.
*   **루트 도메인 (`autosite.kr` 등)**: 절대 건드리지 않습니다! (기존 티스토리 연결 유지)
*   **서브 도메인 추가 (`maza.autosite.kr`)**: `CNAME` 레코드를 추가하고 값에 넷리파이 주소(`[부여받은주소].netlify.app`)를 넣습니다. (클라우드플레어 시 'DNS 전용' 회색 구름)
*   **💡 왜 서브도메인은 CNAME인가요?**: 서브도메인에 CNAME을 쓰면 넷리파이의 글로벌 로드밸런싱(자동 속도 최적화) 혜택을 100% 받아 전 세계 어디서든 1초 만에 로딩됩니다.

### 시나리오 B: 티스토리를 완전히 버리고 '완전 독립' (루트 도메인 이사)
애드센스 승인도 끝났고 티스토리 자체를 버리고 싶다면, 기존 티스토리 연결 레코드를 모두 삭제하고 도메인 전체를 마자 블로그(Zero-IT)로 이사합니다.
*   **기존 티스토리 레코드**: 과감히 삭제합니다.
*   **루트 도메인 (`autosite.kr`)**: 새로 `A` 레코드를 추가하고 IP에 `75.2.60.5`를 넣습니다.
*   **💡 왜 루트 도메인은 A 레코드인가요?**: 국제 DNS 표준 규칙상 루트 도메인(앞에 아무것도 안 붙은 기본 주소)에는 CNAME을 쓸 수 없습니다. 따라서 고정 IP를 가리키는 A 레코드를 쓰는 것이 가장 안전하고 표준적인 방식입니다.

## 🏢 구입처별 상세 설정 방법

### 1️⃣ 가비아 (Gabia)
1. 가비아 로그인 후 **[My가비아] ➡️ [DNS 관리툴]**로 이동합니다.
2. 설정할 도메인의 우측에 있는 **[설정]** 버튼을 클릭합니다.
3. **[레코드 추가]** 버튼을 눌러 아래 두 줄을 차례대로 입력하고 [저장]합니다.
   *   타입: `A`, 호스트: `@`, 값/위치: `75.2.60.5`, TTL: `3600`
   *   타입: `CNAME`, 호스트: `www`, 값/위치: `[본인넷리파이주소].netlify.app.`, TTL: `3600`
   > ⚠️ **주의:** 가비아에서 CNAME 값을 입력할 때는 **맨 끝에 점(.)을 꼭 찍어야 합니다!** (예: `...netlify.app.`)

### 2️⃣ 호스팅케이알 (Hosting.kr)
1. 호스팅케이알 로그인 후 **[나의 서비스] ➡️ [도메인 관리]**로 이동합니다.
2. 해당 도메인을 클릭하고 **[네임서버/DNS]** 탭을 선택합니다.
3. DNS 레코드 설정에서 **[새 레코드 추가]**를 눌러 아래 두 줄을 입력하고 [적용]합니다.
   *   유형: `A`, 이름: `@`, 값: `75.2.60.5`
   *   유형: `CNAME`, 이름: `www`, 값: `[본인넷리파이주소].netlify.app`

### 3️⃣ 카페24 (Cafe24)
1. 카페24 호스팅센터 로그인 후 **[나의 서비스 관리] ➡️ [도메인 관리] ➡️ [DNS 관리]**로 이동합니다.
2. 해당 도메인을 선택하고 **[DNS 관리]** 버튼을 클릭합니다.
3. **[A 레코드 추가]** 및 **[CNAME 레코드 추가]**를 통해 아래와 같이 설정합니다.
   *   A 레코드 ➡️ 호스트명: 공란(비워둠), IP주소: `75.2.60.5`
   *   CNAME 레코드 ➡️ 별칭: `www`, 실제도메인명: `[본인넷리파이주소].netlify.app`

### 4️⃣ 클라우드플레어 (Cloudflare)
1. 클라우드플레어 로그인 후 해당 도메인 프로젝트를 선택합니다.
2. 좌측 메뉴에서 **[DNS] ➡️ [레코드]**로 이동합니다.
3. **[레코드 추가(Add record)]** 버튼을 눌러 아래 두 줄을 입력합니다.
   *   Type: `A`, Name: `@`, IPv4 address: `75.2.60.5`, Proxy status: `Proxied(주황색 구름)`
   *   Type: `CNAME`, Name: `www`, Target: `[본인넷리파이주소].netlify.app`, Proxy status: `Proxied(주황색 구름)`

## ⏳ 설정 후 참고사항
*   DNS 레코드를 변경한 후, 전 세계 인터넷 망에 이 정보가 퍼지려면(DNS 전파) 빠르면 5분, **길게는 최대 24~48시간**까지 소요될 수 있습니다.
*   설정 직후 도메인으로 접속했을 때 사이트가 보이지 않거나 브라우저에서 '안전하지 않음(보안 인증서 오류)'이 뜨더라도 **절대 설정이 잘못된 것이 아닙니다.** 잠시 커피 한 잔 드시고 오시면 넷리파이가 자동으로 보안 인증서(SSL) 발급까지 모두 완료해 줍니다!
