# Maza Extension 가이드

Maza Extension은 사용자의 외부 블로그 플랫폼(예: 티스토리)과 Maza Studio를 매끄럽게 연결해주는 핵심 컴포넌트입니다.

## 1. 목적 (Why do we need this?)

외부 블로그 플랫폼(티스토리, 네이버 블로그 등)은 Open API를 제공하지 않거나, 제공하더라도 일일 발행 한도나 이미지 업로드 기능 등에서 제한적인 경우가 많습니다.
Maza Extension은 유저의 브라우저에서 **DOM 인젝션(DOM Injection)** 방식을 사용하여 사용자가 직접 글을 쓰는 것과 완전히 동일한 플로우로 글을 자동으로 발행해 줍니다. 

이는 다음과 같은 장점을 제공합니다:
- API 제한 우회 및 안정적인 대량 발행
- 플랫폼의 봇 탐지 정책 회피
- 플랫폼 변경 시 유연한 대응 가능

## 2. 퍼블리시 플로우 (Publish Flow)

발행 프로세스는 오토파일럿 엔진에 의해 비동기적으로 스케줄링되며 다음과 같이 작동합니다:

1. **태스크 감지**: 백그라운드 워커가 1분 주기로 Maza Studio 서버를 폴링하여 대기 중인 `PUBLISH_POST` 태스크를 가져옵니다.
2. **에디터 진입**: 익스텐션이 사용자의 브라우저에서 백그라운드 탭을 열고 타겟 블로그의 글쓰기 페이지(`https://[blog_name].tistory.com/manage/post`)로 진입합니다.
3. **콘텐츠 주입**:
   - `제목` 입력칸에 제목 삽입
   - `카테고리` 선택 (존재할 경우)
   - `태그` 입력 및 엔터
   - iframe 또는 shadow DOM 내부의 에디터 캔버스를 찾아 본문 HTML 인젝션
4. **발행/저장**: 티스토리의 하단 **완료** 버튼과 **공개 발행** 버튼을 순차적으로 클릭합니다. (경우에 따라 임시저장 수행)
5. **결과 보고**: 성공 여부 및 발행된 URL을 Maza Studio 서버로 콜백 전송하여 태스크 상태를 `PUBLISHED`로 변경합니다.

## 3. Playwright E2E 테스트 구조

Extension의 발행 엔진은 브라우저 렌더링에 의존하므로, UI가 조금만 변경되어도 동작이 실패할 수 있습니다. 이를 방지하기 위해 `Playwright`를 이용한 E2E(End-to-End) 테스트를 수행합니다.

- **Mock Server 방식**: 실제 티스토리 서버에 부담을 주거나 스팸 글이 발행되는 것을 막기 위해 로컬에 티스토리와 유사한 DOM 구조를 가진 Mock Server(모의 에디터 환경)를 띄워 테스트합니다.
- **주요 테스트 케이스**:
  - `publish.spec.ts`: 제목, 본문, 태그, 카테고리가 제대로 주입되는지 검증
  - 에러 처리 플로우: 에디터 요소를 찾지 못했을 때 우아하게 실패(Graceful Failure) 처리 후 에러 리포트 전송 여부 검증
- **테스트 실행**:
  ```bash
  npx playwright test --config=playwright.no-webserver.config.ts
  ```

> **주의**: 크롬 확장의 경우 백그라운드 서비스 워커(Service Worker)가 유휴 상태일 때 Sleep 모드로 들어갈 수 있으므로, MessageBus 리스너 작성 시 `return true;` 와 같은 비동기 콜백 패턴을 반드시 준수해야 합니다. (자세한 내용은 `CRITICAL_FIXES.md` 참조)
