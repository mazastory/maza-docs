안녕하세요, Maza Studio입니다! 💡

티스토리 블로그를 운영하시다가 나만의 멋진 개인 도메인(autosite.kr 같은!)을 연결하는 순간은 정말 설레죠? 하지만 이때, 기존의 티스토리 주소(zerowork.tistory.com)에서 새 주소로의 '이사'를 검색 엔진에 정확히 알려주지 않으면, 그동안 쌓아 올렸던 소중한 검색 노출 점수(SEO)가 사라지거나 글들이 제대로 노출되지 않을까 걱정하실 수 있습니다.

"도메인 변경 시 무엇을 해야 할까?" "기존 글들은 어떻게 될까?"

이런 IT 초보자분들의 궁금증을 Maza Studio가 쉽고 명쾌하게 해결해 드립니다! 개인 도메인 연결 후 필수 체크리스트와 사이트맵/RSS 처리 방법을 완벽하게 정리했습니다. 지금부터 차근차근 알아볼까요?

1. 🔍 검색 엔진은 새 도메인을 "완전히 새로운 사이트"로 인식합니다!

이것이 개인 도메인 연결 시 가장 중요하게 기억해야 할 핵심 원칙입니다.

기존 티스토리 주소로 20개를 썼든, 100개를 썼든 상관없이, 개인 도메인을 입히는 순간 구글(서치콘솔)과 네이버(서치어드바이저) 입장에서는 **"완전히 새로운, 깡통 사이트가 방금 생겼다!"**고 판단합니다. 😲

따라서, 우리는 이 '새로운 사이트'를 검색 엔진에 다시 소개하고, 마치 처음부터 다시 시작하듯이 모든 검색 엔진 도구에 등록하고 필요한 세팅을 처음부터 다시 진행해야 합니다. 걱정 마세요, Maza Studio와 함께라면 어렵지 않습니다!

2. 📝 기존 글들의 운명은? 사이트맵 및 RSS 반영 완벽 해부!

가장 많이 궁금해하시는 질문 중 하나죠?

Q. 기존에 20개의 글을 작성한 상태에서 개인 도메인을 연결하면, 사이트맵과 RSS에는 21번째 글부터 인식되나요? 기존 글들은 검색에서 누락되는 건 아닌가요?

A. 아닙니다! 걱정 마세요. 기존 글(1~20번째 글)들도 모두 새 도메인으로 완벽하게 인식됩니다! 🎉

개인 도메인을 티스토리에 연결하는 순간, 티스토리 시스템이 똑똑하게도 기존에 작성했던 모든 글들을 자동으로 인식하여 새로운 도메인 기준의 사이트맵과 RSS를 생성해 줍니다.

예를 들어, https://autosite.kr/sitemap.xml에 접속해 보면 다음과 같이 예전 글들도 모두 새 도메인 주소로 깔끔하게 바뀌어 목록에 포함된 것을 확인할 수 있습니다.

  • https://autosite.kr/1 (1번째 글)
  • https://autosite.kr/2 (2번째 글)
  • ...
  • https://autosite.kr/20 (20번째 글)

그러니 도메인 연결 전에 쓴 글이라고 해서 검색에서 누락될까 봐 걱정하실 필요가 전혀 없습니다. 과거의 소중한 글들도 모두 새 도메인 주소로 검색 엔진에 멋지게 노출될 것입니다! ✨

3. ⚙️ 개인 도메인 연결 후 필수 체크리스트 & 설정 가이드

개인 도메인을 성공적으로 연결하셨다면, 이제 검색 엔진이 당신의 새 보금자리를 제대로 찾아오도록 몇 가지 중요한 설정을 다시 해줘야 합니다. 아래 항목들을 순서대로 빠짐없이 진행해 주세요!

① 웹마스터 도구 (Google Search Console / 네이버 서치어드바이저) 재등록

이 단계는 절대 건너뛰시면 안 됩니다! 기존 티스토리 주소로 사이트맵과 RSS를 제출했더라도, 새 개인 도메인 연결 후에는 무조건 '새로운 사이트'로 다시 등록해야 합니다.

  1. Google Search Console 및 네이버 서치어드바이저에 접속하여 '새로운 속성 추가' 기능을 이용해 autosite.kr (또는 여러분의 새 도메인)을 등록합니다.
  2. 도메인에 대한 소유권 확인을 다시 진행합니다. (HTML 메타 태그 추가, DNS 레코드 추가 등 방법을 선택하여 완료합니다.)
  3. 새로 등록한 autosite.kr 속성 메뉴에서 사이트맵(sitemap.xml)과 rss를 새로 제출합니다.

💡 Maza Studio 꿀팁 (주소 이전 기능 잊지 마세요!): 새로운 속성(autosite.kr)을 등록하신 후, 기존 속성(zerowork.tistory.com)의 서치콘솔 설정 메뉴에 가보시면 [주소 변경] 이라는 아주 유용한 기능이 있습니다. 여기서 "내가 새 도메인으로 이사했다!"라고 구글에 신고해 주시면, 기존 티스토리 도메인으로 힘들게 쌓아두었던 검색 노출 점수(SEO)를 새 도메인으로 더 빠르고 안전하게 계승받을 수 있습니다. 놓치지 마세요! 📌

② 구글 애드센스 (Google AdSense) 설정

애드센스로 블로그 수익을 창출하고 계셨다면, 몇 가지 재설정이 필요합니다.

  • 사이트 추가 및 승인: 새 개인 도메인(autosite.kr)을 애드센스의 '사이트' 메뉴에 새로 추가하고 **다시 승인(애드고시)**을 받아야 합니다. 티스토리 도메인으로 승인받은 하위 도메인이 아니기 때문에 별도의 심사가 필요합니다.
  • ads.txt 문제 해결: 최근 티스토리에서는 [수익] 메뉴에서 ads.txt 연동을 편리하게 지원합니다. 개인 도메인 연결 후 이 설정이 정상적으로 연결되어 있는지 반드시 확인해 주세요.

③ 방문자 분석 도구 (Google Analytics 등) 업데이트

블로그 방문자 통계를 확인하는 구글 애널리틱스(GA4) 같은 도구도 업데이트해야 합니다.

  • 구글 애널리틱스(GA4) 관리자 설정에서 해당 속성의 기본 URL을 기존 티스토리 주소에서 새 개인 도메인 주소로 변경해야 합니다. 이렇게 해야 유입 데이터가 정확히 새 도메인 기준으로 측정됩니다.

④ 메타데이터 및 HTML 헤드 (Canonical 태그) 확인

검색 엔진은 똑같은 내용의 페이지가 여러 주소로 존재할 때 어떤 것을 '원본'으로 볼지 혼란스러워하고, 이는 '중복 문서' 페널티로 이어질 수 있습니다. 이를 방지하는 것이 바로 Canonical 태그입니다.

  • 티스토리 관리자 설정에서 개인 도메인을 정상적으로 연결(그리고 보안 접속 인증서 발급까지 완료)하면, 티스토리가 자동으로 HTML <head> 태그 내의 <link rel="canonical" href="..."> 태그를 새 개인 도메인 기준으로 변경해 줍니다.
  • 적용 후 브라우저에서 F12 (개발자 도구)를 눌러 <head> 태그 안에 canonical 주소가 새 도메인으로 잘 설정되어 있는지 한 번 더 확인하시면 더욱 안심할 수 있습니다. ✅

⑤ 스킨 내 하드코딩된 링크 수정

가끔 티스토리 스킨(HTML/CSS) 편집을 통해 메뉴나 로고 이미지, 푸터 등에 기존 티스토리 주소를 직접 적어두시는 경우가 있습니다.

  • 이러한 하드코딩된 링크들이 있다면, 모두 찾아서 새 개인 도메인 주소로 수정해야 합니다.
  • 또한, 본문 글을 작성할 때 삽입했던 이전 주소의 내부 링크들도 여유가 되실 때 새 주소로 수정하시는 것이 장기적인 SEO에 더욱 좋습니다.
