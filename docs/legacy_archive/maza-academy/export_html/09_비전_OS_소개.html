🚀 MAZA Studio: 100점 아키텍처 비전 – 미래형 블로그 운영 OS를 만나다!

안녕하세요, 미래형 블로그 운영의 동반자, MAZA Studio입니다! 👋

저희는 단순히 글을 쓰는 도구를 넘어, 여러분의 블로그를 AI와 함께 전략적으로 성장시키는 **"AI 기반 블로그 운영 워크스페이스(Blog Operation OS)"**를 만들고 있습니다. 오늘 이 글을 통해, MAZA의 핵심 철학과 견고한 아키텍처 설계가 어떻게 여러분의 블로그 경험을 혁신할지 자세히 소개해 드릴게요!

🎯 우리의 궁극적인 목표: 단순한 확장을 넘어선 OS로!

MAZA는 단순한 크롬 익스텐션의 한계를 뛰어넘어, 여러분의 블로그 운영 전반을 책임지는 종합적인 AI 블로그 운영 OS를 지향합니다. 마치 여러분의 블로그에 전담 AI 비서가 생긴 것처럼, 기획부터 발행, SEO 최적화까지 모든 과정을 지능적으로 지원하죠.

🌟 MAZA 아키텍처의 5가지 핵심 원칙

MAZA의 아키텍처는 사용자에게 최고의 안정성과 효율성을 제공하기 위한 명확한 원칙들을 기반으로 설계되었습니다. 이 원칙들은 MAZA가 단순한 도구가 아닌, 강력한 '운영 OS'가 될 수 있도록 뒷받침합니다.

  1. Extension 최소화 (Dumb Bridge 전략) 💡

    • 크롬 익스텐션은 웹 페이지와 서버 간의 '덤 브릿지(Dumb Bridge)' 역할만 수행하도록 설계했습니다. 불필요한 복잡성을 제거하고, 오직 DOM 제어와 사용자 인터랙션에만 집중하여 가볍고 빠르게 동작하도록 합니다.

  2. Server 중심 구조 🧠

    • 블로그 운영의 '뇌'는 서버에 있습니다! AI 로직, 복잡한 상태 관리, 그리고 모든 자동화 작업의 오케스트레이션은 강력한 서버가 전담합니다. 이로써 익스텐션의 부담을 줄이고, 서비스의 안정성과 확장성을 극대화합니다.

  3. Task 기반 자동화 (Polling 방식) 🔄

    • MAZA의 모든 자동화 작업은 '태스크(Task)' 단위로 생성되고 관리됩니다. 안정적인 Polling 방식을 통해 작업의 진행 상황을 서버가 지속적으로 확인하고, 오류 없이 완벽하게 수행될 수 있도록 보장합니다.

  4. 플랫폼 완전 분리 (Adapter Pattern 도입) 🧩

    • Tistory, WordPress, Blogspot 등 다양한 블로그 플랫폼에 유연하게 대응하기 위해 **어댑터 패턴(Adapter Pattern)**을 도입했습니다. 각 플랫폼은 독립적인 어댑터를 통해 MAZA와 연동되어, 새로운 플랫폼 추가도 훨씬 쉽고 빠르게 이뤄질 수 있습니다.

  5. 안정성 최우선 (MV3 친화적 Stateless 구조) 🛡️

    • Chrome Manifest V3(MV3) 환경에 완벽하게 친화적인 Stateless(무상태) 구조를 채택했습니다. 이는 익스텐션의 안정성을 높이고, 예측 불가능한 오류를 최소화하여 사용자 경험을 최상으로 유지하는 데 기여합니다.

🏗️ MAZA의 구조적 혁신: 강력하고 유연한 설계

MAZA의 아키텍처는 사용자 경험을 최우선으로 고려하며, 각 구성 요소가 명확한 역할을 수행하도록 설계되었습니다.

🏢 Dashboard (Workspace) – 여러분의 블로그 사령탑

MAZA의 대시보드는 블로그 운영의 모든 것을 한눈에 보고 관리할 수 있는 여러분의 개인 워크스페이스입니다.

  • UI Concept: Arc Browser의 미니멀리즘, Linear의 효율성, Notion의 유연성을 결합한 직관적이고 강력한 인터페이스를 자랑합니다.
  • 핵심 페이지: Projects (프로젝트 관리), Blogs (연동된 블로그 관리), Schedules (예약 발행 및 작업 일정), SEO (검색 엔진 최적화), Logs (모든 작업 기록), AI Assistant (AI 비서 기능) 등 블로그 운영에 필요한 모든 기능이 집약되어 있습니다.

💉 Extension (Dumb Bridge) – 가볍고 민첩한 현장 요원

저희 익스텐션은 이름 그대로 '덤 브릿지' 역할에 충실합니다. 웹 페이지에서 필요한 작업을 정확하고 빠르게 수행하죠.

  • 역할: 웹 페이지의 DOM 제어, 사용자 로그인 유지, 콘텐츠 입력, 이미지/파일 업로드, 포스트 발행 및 작업 결과 반환 등 최소한의 필수 기능에 집중합니다.
  • 제거 항목: 불필요한 WebSocket Persistence, Queue Manager, 복잡한 Workflow Engine, 그리고 골치 아픈 Reconnect Hell을 완전히 제거하여 익스텐션을 극도로 가볍고 안정적으로 만들었습니다.

🧠 Server (Command Center) – MAZA의 강력한 두뇌

MAZA 서버는 모든 AI 로직과 핵심 운영을 담당하는 명령 센터입니다.

  • 역할: 고품질 AI 콘텐츠 생성, 모든 Task의 생성 및 관리, 정교한 예약 발행, 고급 SEO 자동화, 그리고 모든 서비스의 상태 및 로그 관리까지 책임집니다.
  • 기술 스택:
    • Fastify: 초고속 웹 프레임워크로, 모든 요청을 빠르고 효율적으로 처리합니다.
    • PostgreSQL: 안정적인 관계형 데이터베이스로, 모든 핵심 데이터를 안전하게 저장합니다.
    • Prisma: ORM(Object-Relational Mapping) 툴로, 데이터베이스와의 상호작용을 간편하고 안전하게 만듭니다.
    • Redis (BullMQ): 고성능 인메모리 데이터 저장소 및 메시지 브로커로, 복잡한 작업 큐(Queue)를 효율적으로 관리하여 대규모 자동화 작업을 안정적으로 처리합니다.

🔌 Platform Adapters – 블로그 플랫폼 간의 만능 연결고리

MAZA는 다양한 블로그 플랫폼에 대한 지원을 유연하게 확장할 수 있습니다.

  • 구조: Tistory, Wordpress, Blogspot 등 각 플랫폼에 대한 어댑터를 독립적으로 개발하여, 각 플랫폼의 특성에 최적화된 방식으로 연동됩니다.
  • Selector Registry: 플랫폼별로 필요한 웹 요소(버튼, 입력 필드 등)의 셀렉터를 중앙 집중식으로 관리하여, 유지보수 및 확장성을 크게 향상시킵니다.

✨ 핵심 UX/UI: 에이전트와 함께하는 몰입형 경험

MAZA는 단순한 기능을 넘어, 사용자에게 특별한 경험을 선사합니다.

  • 페르소나: 모든 작업에서 "Agent" 느낌을 강조합니다. 예를 들어, MAZA is generating..., MAZA is optimizing SEO...와 같은 메시지를 통해, 마치 전담 AI 비서가 여러분의 블로그를 위해 일하고 있다는 몰입감과 신뢰감을 선사합니다.
