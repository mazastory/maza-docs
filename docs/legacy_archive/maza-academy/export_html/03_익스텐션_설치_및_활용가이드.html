🚀 Maza Extension: 당신의 블로그 자동화를 위한 강력한 연결고리!

안녕하세요, 'Maza Studio' 블로그 독자 여러분! 💡 블로그 운영, 특히 여러 플랫폼에 글을 발행하는 작업은 생각보다 많은 시간과 노력을 필요로 합니다. Maza Studio는 이 과정을 혁신적으로 자동화하여 여러분의 소중한 시간을 절약해 드립니다. 그리고 이 마법 같은 자동화의 핵심에는 바로 Maza Extension이 자리 잡고 있습니다!

오늘은 Maza Extension이 어떻게 여러분의 외부 블로그(예: 티스토리)와 Maza Studio를 매끄럽게 연결해주는지, 그 놀라운 기술과 작동 방식에 대해 자세히 알아보겠습니다. 🚀

🎯 1. 왜 Maza Extension이 필요한가요? (Why do we need this?)

많은 분들이 티스토리, 네이버 블로그 같은 외부 플랫폼의 API 한계에 부딪혀 답답함을 느끼셨을 거예요.

  • Open API 부재: 아예 Open API를 제공하지 않는 플랫폼도 많습니다.
  • 제한적인 API: 제공하더라도 '하루 발행 한도', '이미지 업로드 기능 부재' 등 다양한 제약이 따르곤 합니다.

이런 한계를 극복하기 위해 Maza Extension은 특별한 방식을 사용합니다. 바로 DOM 인젝션 (DOM Injection) 방식인데요! 🌐

DOM 인젝션이란? 🤔 여러분의 웹 브라우저에서 마치 여러분이 직접 키보드를 치고 마우스를 클릭하는 것처럼, 글쓰기 페이지의 특정 위치에 제목, 본문, 카테고리 등을 자동으로 삽입하고 버튼을 클릭하는 기술을 말합니다. 마치 숙련된 디지털 비서가 대신 작업을 처리해주는 것과 같아요!

이 방식 덕분에 Maza Extension은 다음과 같은 강력한 장점을 제공합니다:

  • API 제한 우회 및 안정적인 대량 발행: 플랫폼의 공식 API 제약을 받지 않고, 원하는 만큼 안정적으로 글을 발행할 수 있습니다.
  • 플랫폼의 봇 탐지 정책 회피: 사람처럼 동작하므로, 봇으로 오인될 위험이 현저히 낮습니다.
  • 플랫폼 변경 시 유연한 대응 가능: 플랫폼의 UI가 조금 바뀌더라도, 코드 업데이트를 통해 빠르게 대응할 수 있습니다.

⚙️ 2. 퍼블리시 플로우: 당신의 블로그 글이 발행되는 과정 (Publish Flow)

자, 그럼 Maza Extension이 실제로 여러분의 글을 어떻게 발행하는지, 그 자동화 프로세스를 단계별로 살펴볼까요? 이 모든 과정은 Maza Studio의 오토파일럿 엔진에 의해 비동기적으로(동시에 여러 작업을 처리하며) 스케줄링됩니다.

1단계: 태스크 감지 (Task Detection) 🕵️‍♀️

  • Maza Extension의 백그라운드 워커는 1분 주기로 Maza Studio 서버를 꼼꼼히 확인합니다.
  • 만약 대기 중인 PUBLISH_POST (글 발행) 태스크를 발견하면 즉시 가져옵니다.

2단계: 에디터 진입 (Editor Entry) 📝

  • 익스텐션은 여러분의 브라우저에서 백그라운드 탭을 조용히 열고, 목표 블로그의 글쓰기 페이지(https://[blog_name].tistory.com/manage/post)로 이동합니다.

3단계: 콘텐츠 주입 (Content Injection) ✍️

  • 제목: 글쓰기 에디터의 제목 입력칸에 글의 제목을 정확히 삽입합니다.
  • 카테고리: 설정된 카테고리가 있다면 해당 카테고리를 선택합니다.
  • 태그: 태그를 입력하고 엔터 키를 눌러 적용합니다.
  • 본문: iframe 또는 shadow DOM 내부에 있는 에디터 캔버스를 찾아, 본문 HTML 콘텐츠를 깔끔하게 삽입합니다. (이 복잡한 과정도 Extension이 알아서 처리해요!)

4단계: 발행/저장 (Publish/Save) ✅

  • 티스토리 글쓰기 페이지 하단의 완료 버튼과 공개 발행 버튼을 순서대로 클릭하여 글을 발행합니다. (때로는 임시 저장 기능도 활용합니다.)

5단계: 결과 보고 (Result Report) 📊

  • 글 발행이 성공했는지 여부와 발행된 글의 URL을 Maza Studio 서버로 다시 전송합니다.
  • 이후 태스크 상태는 PUBLISHED로 변경되어, 여러분은 Maza Studio에서 발행 결과를 바로 확인할 수 있습니다.

🧪 3. Playwright E2E 테스트 구조: 튼튼한 발행 엔진을 위한 노력!

Maza Extension은 브라우저의 화면 렌더링(UI)에 의존하여 작동하기 때문에, 블로그 플랫폼의 UI가 조금만 변경되어도 발행이 실패할 수 있습니다. 이러한 만일의 사태를 방지하고 안정성을 극대화하기 위해, 우리는 Playwright를 이용한 E2E (End-to-End) 테스트를 꾸준히 수행합니다.

Playwright란? 🛠️ 웹 애플리케이션의 사용자 인터페이스를 자동으로 테스트하는 도구로, 실제 사용자가 웹사이트를 사용하는 것처럼 시뮬레이션할 수 있게 해줍니다.

  • Mock Server 방식: 실제 티스토리 서버에 무리를 주거나 테스트 과정에서 불필요한 스팸 글이 발행되는 것을 막기 위해, 우리는 로컬 환경에 티스토리와 거의 동일한 DOM 구조를 가진 **Mock Server (모의 에디터 환경)**를 띄워 테스트를 진행합니다.
  • 주요 테스트 케이스:
    • publish.spec.ts: 제목, 본문, 태그, 카테고리가 의도한 대로 정확하게 주입되는지 검증합니다.
    • 에러 처리 플로우: 에디터 요소를 찾지 못했을 때 우아하게 실패(Graceful Failure) 처리 후, 정확한 에러 리포트를 Maza Studio 서버로 전송하는지 검증합니다.
  • 테스트 실행 명령어:
    npx playwright test --config=playwright.no-webserver.config.ts

📌 주의사항: 크롬 확장 프로그램의 경우 백그라운드 서비스 워커(Service Worker)가 오랫동안 유휴 상태일 때 Sleep 모드로 들어갈 수 있습니다. 따라서 MessageBus 리스너를 작성할 때는 return true; 와 같은 비동기 콜백 패턴을 반드시 준수해야 합니다. (더 자세한 내용은 CRITICAL_FIXES.md를 참고해주세요!)

Maza Extension은 여러분의 블로그 자동화를 위한 강력한 엔진이자 든든한 조력자입니다. 복잡하고 반복적인 발행 작업은 이제 Maza Studio에 맡기고, 여러분은 오직 양질의 콘텐츠 생산에만 집중하세요! ✍️✨

👉 Maza Extension에 대한 더 깊이 있는 정보는 마자 스튜디오 공식 문서(익스텐션 가이드)에서 확인하실 수 있습니다!

Maza Studio와 함께라면, 여러분의 블로그 운영이 더욱 스마트하고 효율적으로 바뀔 것입니다! 감사합니다. 😊
