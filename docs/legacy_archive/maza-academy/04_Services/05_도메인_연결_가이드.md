# 🚀 나만의 도메인으로 Maza Studio 사이트를 빛내주세요! (DNS 연결 완벽 가이드)

🎉 드디어 Maza Studio에서 빛나는 나만의 웹사이트를 완성하셨습니다! 이제 마지막 한 끗, 바로 **도메인 연결**입니다. 여러분이 구매하신 도메인을 실제 사이트와 연결하면, 기억하기 쉬운 주소로 전 세계에 여러분의 멋진 콘텐츠를 선보일 수 있습니다.

이 과정은 생각보다 훨씬 간단해요! 도메인을 구입하신 사이트(가비아, 호스팅케이알 등)에 접속하셔서 **DNS 레코드**를 딱 두 줄만 추가해 주시면 끝이랍니다. IT 초보자분들도 걱정 마세요! Maza Studio가 쉽게 단계별로 안내해 드릴게요. 😉

---

## 🎯 도메인 연결의 핵심! 필수 DNS 레코드 정보

도메인을 어디서 구매하셨든 상관없이, 우리가 입력해야 할 핵심 정보는 아래 두 가지 시나리오로 나뉩니다.

### 1. 웹사이트의 '루트 도메인'을 연결할 경우 (예: `mysite.com`)

가장 일반적인 형태인데요, 도메인 이름만으로 바로 사이트에 접속하게 하고 싶을 때 사용합니다. `www`를 붙이지 않아도 접속되고, `www`를 붙여도 접속되도록 두 줄의 레코드가 필요해요.

*   **1단계: A 레코드 추가 (루트 도메인 연결)**
    *   **타입/종류:** `A` (IP 주소로 직접 연결한다는 의미)
    *   **이름/호스트:** `@` (또는 빈칸, 또는 여러분의 도메인 이름 `mysite.com`)
    *   **값/IP주소:** `75.2.60.5` (넷리파이의 고정 IP 주소입니다.)
    
*   **2단계: CNAME 레코드 추가 (www 접속 지원)**
    *   **타입/종류:** `CNAME` (다른 주소로 별명을 붙여준다는 의미)
    *   **이름/호스트:** `www`
    *   **값/목적지:** `[부여받은 넷리파이 주소].netlify.app` (예: `happy-blog-12345.netlify.app`)

### 2. '서브 도메인'을 연결할 경우 (예: `blog.mysite.com`)

기존에 운영 중인 웹사이트나 서비스가 있고, 그 하위에 새로운 블로그나 페이지를 추가하고 싶을 때 유용합니다. 서브 도메인은 IP 주소(A 레코드)가 필요 없고, CNAME 레코드 딱 1줄만 추가하면 됩니다.

*   **CNAME 레코드 추가**
    *   **타입/종류:** `CNAME`
    *   **이름/호스트:** `blog` (원하는 서브도메인 이름, 예: `shop`, `project` 등)
    *   **값/목적지:** `[부여받은 넷리파이 주소].netlify.app`

## 🚀 [심화] 티스토리 사용자를 위한 마자 독립 블로그 확장 및 이사 전략

이미 애드센스 승인을 받은 개인 도메인 티스토리 블로그(`autosite.kr`)를 운영 중이신가요? 💡 티스토리의 한계를 벗어나 마자 독립 블로그로 확장하거나 완전히 이사(독립)할 때의 **현명한 DNS 설정 전략**을 알려드립니다!

### 시나리오 A: 티스토리와 독립 블로그 '동시 운영' (서브도메인 확장)

가장 추천하는 방법입니다! 💰 기존 티스토리 본진(`autosite.kr`)은 그대로 살려두고, 새로운 서브도메인(`maza.autosite.kr`)만 Maza Studio 독립 블로그로 세팅하여 수익 파이프라인을 2배로 늘릴 수 있습니다.

*   **루트 도메인 (`autosite.kr` 등)**: **절대 건드리지 마세요!** 기존 티스토리 연결을 그대로 유지합니다.
*   **서브 도메인 추가 (`maza.autosite.kr`)**: `CNAME` 레코드를 추가하고 값에 여러분의 넷리파이 주소(`[부여받은주소].netlify.app`)를 입력합니다.
    *   📌 **클라우드플레어 사용자 주의:** 설정 시 'DNS 전용' (회색 구름)으로 두셔야 합니다.
*   > 💡 **왜 서브도메인은 CNAME인가요?**
    > 서브도메인에 CNAME을 사용하면 넷리파이의 강력한 **글로벌 로드밸런싱** (자동 속도 최적화) 혜택을 100% 받을 수 있습니다. 전 세계 어디서든 여러분의 사이트가 1초 만에 로딩되는 마법을 경험하실 거예요!

### 시나리오 B: 티스토리를 완전히 버리고 '완전 독립' (루트 도메인 이사)

애드센스 승인도 마쳤고, 이제 티스토리를 완전히 벗어나 Maza Studio 독립 블로그(Zero-IT)로 갈아타고 싶으시다면, 이 전략을 사용하세요.

*   **기존 티스토리 레코드**: 과감하게 **모두 삭제**합니다!
*   **루트 도메인 (`autosite.kr`)**: 위 **'1. 루트 도메인 연결'** 섹션에서 설명한 대로 새로운 `A` 레코드와 `CNAME` 레코드를 추가합니다.
    *   `A` 레코드: IP 주소 `75.2.60.5`
    *   `CNAME` 레코드: `www`에 넷리파이 주소(`[부여받은주소].netlify.app`) 연결
*   > 💡 **왜 루트 도메인은 A 레코드인가요?**
    > 국제 DNS 표준 규칙상 루트 도메인 (앞에 `www`나 `blog` 같은 접두어가 붙지 않은 기본 주소)에는 CNAME을 직접 사용할 수 없습니다. 따라서 고정 IP를 가리키는 `A` 레코드를 사용하는 것이 가장 안전하고 표준적인 방식입니다.

## 🏢 도메인 구입처별 상세 설정 방법 (캡처 없이도 OK!)

이제 여러분이 도메인을 구매하신 사이트에서 위에서 설명한 DNS 레코드를 실제로 추가하는 방법을 알아볼게요.

### 1️⃣ 가비아 (Gabia) 사용자 가이드

1.  가비아 웹사이트에 로그인 후, 상단 메뉴에서 **[My가비아] ➡️ [DNS 관리툴]**로 이동합니다.
2.  설정할 도메인의 우측에 있는 **[설정]** 버튼을 클릭합니다.
3.  **[레코드 추가]** 버튼을 눌러 아래 두 줄의 레코드를 차례대로 입력하고 [저장]합니다.
    *   타입: `A`, 호스트: `@`, 값/위치: `75.2.60.5`, TTL: `3600`
    *   타입: `CNAME`, 호스트: `www`, 값/위치: `[본인넷리파이주소].netlify.app.`, TTL: `3600`
    > ⚠️ **가비아 CNAME 입력 시 주의:**
    > CNAME 값(`[본인넷리파이주소].netlify.app`)을 입력할 때 **맨 끝에 점(.)을 꼭 찍어야 합니다!** (예: `...netlify.app.`) 이 점 하나가 성공 여부를 좌우할 수 있어요!

### 2️⃣ 호스팅케이알 (Hosting.kr) 사용자 가이드

1.  호스팅케이알 웹사이트에 로그인 후, **[나의 서비스] ➡️ [도메인 관리]**로 이동합니다.
2.  해당 도메인을 클릭하고 **[네임서버/DNS]** 탭을 선택합니다.
3.  DNS 레코드 설정에서 **[새 레코드 추가]**를 눌러 아래 두 줄을 입력하고 [적용]합니다.
    *   유형: `A`, 이름: `@`, 값: `75.2.60.5`
    *   유형: `CNAME`, 이름: `www`, 값: `[본인넷리파이주소].netlify.app`

### 3️⃣ 카페24 (Cafe24) 사용자 가이드

1.  카페24 호스팅센터에 로그인 후, **[나의 서비스 관리] ➡️ [도메인 관리] ➡️ [DNS 관리]**로 이동합니다.
2.  해당 도메인을 선택하고 **[DNS 관리]** 버튼을 클릭합니다.
3.  **[A 레코드 추가]** 및 **[CNAME 레코드 추가]**를 통해 아래와 같이 설정합니다.
    *   A 레코드 ➡️ 호스트명: 공란(비워둠), IP주소: `75.2.60.5`
    *   CNAME 레코드 ➡️ 별칭: `www`, 실제도메인명: `[본인넷리파이주소].netlify.app`

### 4️⃣ 클라우드플레어 (Cloudflare) 사용자 가이드

1.  클라우드플레어에 로그인 후, 해당 도메인 프로젝트를 선택합니다.
2.  좌측 메뉴에서 **[DNS] ➡️ [레코드]**로 이동합니다.
3.  **[레코드 추가(Add record)]** 버튼을 눌러 아래 두 줄을 입력합니다.
    *   Type: `A`, Name: `@`, IPv4 address: `75.2.60.5`, Proxy status: `Proxied` (주황색 구름 켜짐)
    *   Type: `CNAME`, Name: `www`, Target: `[본인넷리파이주소].netlify.app`, Proxy status: `Proxied` (주황색 구름 켜짐)
    > 💡 **클라우드플레어 팁:**
    > 'Proxy status'를 `Proxied` (주황색 구름)로 설정하면, 클라우드플레어의 CDN(콘텐츠 전송 네트워크)과 보안 기능 혜택을 최대로 누릴 수 있습니다!

## ⏳ 설정 후 참고사항: 잠시만 기다려주세요!

DNS 레코드를 변경한 후, 이 정보가 전 세계 인터넷 망에 전파되려면 시간이 필요합니다 (이를 **DNS 전파**라고 부릅니다).
*   빠르면 5분 이내에 반영될 수도 있지만, **길게는 최대 24~48시간**까지 소요될 수 있습니다.
*   설정 직후 도메인으로 접속했을 때 사이트가 보이지 않거나 브라우저에서 '안전하지 않음(보안 인증서 오류)'이라는 메시지가 뜨더라도 **절대 설정이 잘못된 것이 아닙니다.** 잠시 커피 한 잔 마시거나 다른 일을 하고 오시면 넷리파이가 자동으로 보안 인증서(SSL) 발급까지 모두 완료해 줄 거예요! ☕

Maza Studio는 여러분의 성공적인 웹사이트 운영을 언제나 응원합니다! 궁금한 점이 있다면 언제든지 문의해주세요.
