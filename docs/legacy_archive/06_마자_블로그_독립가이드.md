# 🚀 Maza Studio 블로그, 이제 완전한 '내 것'으로! 데이터 독립 및 이관 완벽 가이드

안녕하세요, 블로그 운영에 진심인 여러분! Maza Studio 팀입니다. 😊

SaaS(Software as a Service) 서비스를 이용하면서 가장 큰 걱정 중 하나가 바로 "데이터 종속성(Lock-in)" 아닐까요? "혹시 구독을 해지하면 소중히 작성한 내 블로그 글들이 모두 사라지는 건 아닐까?" 이런 불안감, 저희 Maza Studio는 누구보다 잘 이해하고 있습니다.

그래서 저희는 **고객님의 데이터를 완벽하게 소유하고, 블로그를 완전히 독립시킬 수 있는 특별한 시스템**을 준비했습니다! 💡 이제 Maza Studio 구독이 종료되더라도 걱정 마세요. 여러분은 본인의 데이터베이스(Supabase)를 통해 평생 무료로 Maza 블로그를 운영하고, 내장된 에디터로 글을 계속 작성할 수 있습니다. 여러분의 콘텐츠는 언제나 여러분의 것입니다! 💖

## 🙋‍♀️ 혹시 이런 점이 궁금하셨나요? (FAQ)

블로그 독립에 대해 자주 궁금해하시는 질문들을 모아봤어요!

### Q. Maza Studio 구독을 해지하면 제 블로그와 글들은 모두 삭제되나요?
**A. 절대 그렇지 않습니다!** 🥳
Maza Studio는 고객님의 소중한 데이터가 언제나 여러분의 소유임을 강조합니다. 언제든 **원클릭으로 데이터를 개인 데이터베이스로 완벽하게 이전**할 수 있어요.

한번 독립된 Maza 블로그에는 **평생 무료로 사용할 수 있는 '자체 수동 글쓰기 에디터'**가 내장되어 있답니다. 따라서, 구독이 종료되어도 예쁜 Maza 블로그 템플릿은 그대로 유지하고, 영구적으로 블로그를 소유하며 새로운 글을 발행하거나 기존 글을 수정할 수 있습니다. 정말 멋지지 않나요? ✨

### Q. 블로그 이관 후에는 글쓰기를 어떻게 할 수 있나요?
**A. 독립된 블로그에 내장된 전용 에디터를 사용하시면 됩니다!** ✍️
데이터 이관이 완료된 블로그는 이제 여러분의 독립적인 도메인에서 운영됩니다. `<내 블로그 도메인>/admin` 경로로 접속하시면, 본인의 데이터베이스와 직접 연결된 관리자 대시보드가 나타납니다. 여기서 기존 글을 쉽고 편리하게 수정/삭제할 수 있고, **실시간 미리보기(Live Preview)가 지원되는 마크다운(Markdown) 에디터**로 새로운 글을 발행할 수 있습니다. 마치 나만의 작은 편집실이 생긴 것과 같죠!

## 🚀 내 블로그 완전 독립, 3단계로 끝내기!

Maza Studio에서 어렵게 작성한 글들이 완벽하게 여러분의 소유가 되는 과정은 아주 간단해요. 지금부터 차근차근 따라 해 볼까요?

Maza Studio 대시보드의 `[내 사이트(Site Detail)]` 화면 우측 상단에 있는 **[내 블로그 독립 (데이터 내보내기)]** 버튼을 누르면, 아래 절차들을 안내받으실 수 있습니다.

### 1단계: 💖 무료 데이터베이스(Supabase) 생성

이 단계에서는 여러분의 소중한 블로그 데이터가 영원히 보관될 '집'을 만드는 과정입니다.

1.  먼저 [Supabase](https://supabase.com)에 접속하여 가입하고, **무료 프로젝트를 하나 생성**해 주세요.
2.  이 Supabase 프로젝트가 앞으로 고객님의 블로그 데이터가 영구적으로 보관될 안전한 공간이 될 거예요! 🔐

### 2단계: 💾 테이블 생성 및 데이터 백업

이제 여러분의 블로그 구조를 만들고, 그동안 Maza Studio에서 작성했던 모든 글을 가져올 시간입니다.

1.  Maza Studio 팝업에서 **[DB 스키마 다운로드 (SQL)]** 버튼을 눌러 스크립트 파일을 받아주세요. 이 파일은 여러분의 블로그 데이터를 정리할 틀을 만들어줍니다.
2.  생성하신 Supabase의 'SQL Editor' 메뉴에 다운로드한 SQL 코드를 붙여넣고 실행하여 `sites`와 `posts` 테이블을 생성합니다.
3.  다시 Maza Studio 팝업에서 **[전체 게시글 백업 (JSON)]** 버튼을 눌러 지금까지 작성한 모든 글을 다운로드하세요. 이 파일은 여러분의 노력이 담긴 보물 같은 존재죠! 💎
4.  다운로드한 JSON 파일을 본인의 Supabase에 Import 합니다. 이제 모든 글이 여러분의 데이터베이스로 옮겨졌습니다!

### 3단계: ⚙️ 블로그(Netlify) 연결 정보 변경

마지막으로, 여러분의 블로그 웹사이트가 여러분의 새 데이터베이스를 바라보도록 설정해 줄 거예요.

1.  [Netlify](https://app.netlify.com) 대시보드에 로그인하여 여러분의 블로그 프로젝트에 접속합니다.
2.  `Site Configuration > Environment variables` 메뉴로 이동합니다.
3.  다음 두 가지 환경변수를 **방금 생성한 본인 Supabase의 정보로 교체**해 주세요:
    *   `PUBLIC_SUPABASE_URL` = 나의 Supabase Project URL
    *   `PUBLIC_SUPABASE_ANON_KEY` = 나의 Supabase API Key

**🎉 축하합니다!**
이제 여러분의 블로그는 Maza Studio와 완벽하게 분리되어 **100% 고객님 소유의 블로그**가 되었습니다. `<내도메인>/admin` 에 접속하여 자유롭게 글을 쓰고, 나만의 콘텐츠를 마음껏 펼쳐보세요! 🚀
